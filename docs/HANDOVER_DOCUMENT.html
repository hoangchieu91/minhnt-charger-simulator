1. Tổng Quan Hệ Thống

Hệ thống trạm sạc Minhnt là một nền tảng thiết bị IoT Công nghiệp, bao gồm kiến trúc phân tán 2 lớp (Master-Slave) nhằm đảm bảo sự ổn định của kết nối điện, bảo vệ chập cháy ở mức phần cứng và đồng thời chịu tải kết nối đám mây không mất dữ liệu ngay cả khi đứt mạng.

2. Kiến Trúc Vật Lý & Phần Cứng

ESP32 Gateway (Master)

  • MCU: ESP32-WROVER-B (Dual-core, 240MHz)
  • Bộ nhớ: 8MB Flash (Custom Partitions), 4MB PSRAM
  • Mạng Chính: IC PHY LAN8742A qua giao tiếp RMII (Dùng cáp LAN). Option dự phòng WiFi qua nút nhấn GPIO34 (ấn 5 giây).
  • Tuyế RS485: Dùng RS485 Cổng 2 (UART2: TX=GPIO15, RX=GPIO36, DE=GPIO13) do Cổng 1 bị lỗi thiết kế.
  • Ngoại vi: DS1307 RTC, Hệ đèn báo LED SOS.

STM32 Board (Slave)

  • MCU: STM32F103C8T6
  • Đầu vào Analog: PA0 (Đo chênh lệch NTC20K giải nhiệt).
  • Điều khiển Relay: PB0, PB1 (Khóa chéo Interlock Sạc/Ổ cắm), PB10 (Quạt), PB11 (Khóa cửa).
  • Giao tiếp: PA9/10 (Modbus tới Master), PA2/3 (DLT645 tới Công tơ điện).
  • DIP Switch: Set địa chỉ vòng Modbus (1-7) tại PB5, PB6, PB7.

3. Firmware Cấp Độ Thiết Bị (STM32)

Trọng tâm ở cấp điểm sạc là Machine State (FSM) và kiến trúc phần mềm vòng lặp trống phi tắc nghẽn (Non-blocking). Được quy hoạch nghiêm ngặt làm 4 Layer.

L1

Hardware Abstraction (HAL/LL)

Do CubeMX tự sinh: Thư mục Drivers/, điều khiển thanh ghi gốc vi điều khiển cấp thấp nhất.

L2

Board Support Package (BSP)

Khởi tạo Clock, ngoại vi tại `main.c`, `stm32f1xx_it.c`. Chỉ chứa ngắt trống để ném dữ liệu vào Ring Buffer.

L3

Middleware & Logic Modules

Thư mục Modules/ (`ntc20k.c`, `modbus_slave.c`, `relay_ctrl.c`). Chú ý: relay_ctrl.c ép cứng khóa chéo, không cho phép cấp nguồn nếu relay kia đang đóng (dead-time 100ms).

L4

Application (Main State Machine)

Tập tin app_main.c. Vòng FSM 5 vòng trạng thái quyết định sinh sát phiên sạc: INIT → IDLE → STANDBY → CHARGING → FINISH / ERROR.

⚠️
An Toàn Thiết Bị (Safety Critical)

Lớp STM32 sẽ TỰ ÐỘNG nhảy trạng thái ERROR cắt cầu dao sạc (Disconnect Relay) tự chủ mà không cần chờ Master quyết định nếu: Nhiệt độ NTC > 70°C, Giám sát cửa phát hiện cửa mở (DOOR SENSOR), Cảnh báo phá hoại, hoặc DLT645 báo công suất tiêu thụ vượt Max_Power_W tức thời/110% định mức.

FreeRTOS Task Map

Mã nguồn chạy song song 7 Tasks. Tính năng mạng và giao thức được cô lập.

Core 1 (Real-Time Modbus)

  • task_modbus_polling (1s): Đọc FC04, detect thay đổi.
  • task_modbus_heartbeat (3s): Ghi FC06 xuống slave xác thực kết nối sống.

Core 0 (Networking & Storage)

  • task_mqtt_publish: Bắn gói tin Event Driven.
  • task_mqtt_subscribe: Giải nén JSON từ broker.
  • task_webserver: Phục vụ HTTP/WebSocket.
  • task_time_sync: NTP & RTC DS1307.
  • task_system_monitor: WDT Watchdog feed.

Bắn mỗi 30s khi sạc (Hoặc mỗi 0.1kWh). Chứa Data V, A, W, kWh, Tần số, Nhiệt độ nội tại trụ sạc.

7. Tính Năng Mũi Nhọn: Tự Khôi Phục & Store-and-Forward

🛡️
Cơ Chế Binary Block 64 bytes - LittleFS Queue

Khi cáp quang bị cắt, Mạng rớt: MQTT Task lập tức dừng publish. Tất cả TLM / EVENT Modbus được serialize thành các khối nhị phân cô đặc chỉ 64 Byte (Tiết kiệm vòng đời write flash so với ghi kiểu Text JSON). Ghi vào slot SLOT_PENDING. Tổng sức chứa 28,672 Records.

Khôi phục Queue Replay

Ngay khi MQTT EVENT_CONNECTED nảy cờ. Thuật toán Replay quét LittleFS, dựng lại chuỗi JSON nguyên bản kèm các sequence tăng đơn điệu thời gian thực, gửi tuần tự lên Broker trong Background. Giải phóng bằng SLOT_PUBLISHED đè bộ nhớ tránh phân mảnh.

Bảo vệ Boot-Loop Flash

Trong môi trường sụt điện (Brownout EMI), NVS Flash rất dễ mòn wear-out chết phần cứng. Có cơ chế chặn: NẾU uptime boot dưới 30s xảy ra 5 lần liên tiếp -> Bật Safe Mode (Sos Led), tắt toàn bộ Network, chỉ chạy Local UI để Fix, dứt khoát KHÔNG ghi log nữa để bảo vệ ESP.

Layer Xác Thực JSON Lệnh Tới: Trước khi thực thi bất kỳ lệnh start_charge nào từ Backend trả về, Core ESP phải qua 6 Tầng Guard (Null payload -> Bad Json -> Field check -> Stale Timestamp so với RTC -> Rate Limiting -> ID Out of bounds Bounds Range).

8. Giao Diện Quản Trị Cục Bộ (Local UI)

ESP32 chạy một Web Server siêu nhẹ ngay tại IP tĩnh Gateway phục vụ Kỹ thuật viên bảo trì. Mã Nguồn Web được nén Vue/Vite siêu nhẹ nằm trong phân vùng Spiffs.

  • Cổng giám sát Live WebSocket: Đọc trực tiếp thanh ghi không qua trạm trung chuyển Internet (Low Latency < 50ms). Liệt kê mảng tất cả các trạm đang kết nối.
  • Network Setup: Điền mật khẩu MQTT, tĩnh IP, Set Master ID trực tiếp vào bộ nhớ NVS.
  • Công Cụ Thợ Điện: Cho phép đụng vào Coils để Test mở khóa cửa thay vì cần có App App User, Ping cổng Cloud để xác định DNS có sống không.
